Enumerable Cycle

3/1/2015

The Enumerable module is one of the most powerful modules available to ruby.

Many of the major ruby classes, including Array and Hash, mix in the enumerable module by default in order to make working with collections of objects much easier. Enumerable works by essentially defining a list of objects, and then setting an instruction for each item in said list. For this reason, in order to make an enumerable class of your own, you'll need to define a method.

Object.each { yield }

The above will instruct what you want to do with each element in your collection upon iteration.

One interesting enumerable method that I'd like to talk about is the Enumerable#cycle method

[1,2,3,20,3].cycle(2){|x| x}

Cycle can be thought of as multiple .each iterations through a collection. If you want to iterate through a collection 2 times, rather than performing an action with each, ending a loop, and then defining another action to be performed with each immediately afterwards I can use a single cycle method wherein I pass in the argument 2 to signify how many times I want to perform my iteration.


Cycle can be used to manually create the combinations and permutation of a specific ordering, which makes it extremly valuable for creating algorithmic sequences and preventing unnecessary code bloat. The downside of using cycle can be that you can easily neglect to define when to break out of a cycle unless you either pass in an integer argument as a parameter or you define within your block when to appropriately break.

Ruby's enumerable class has many handy methods that greatly reduce the amount of work the engineer has to manage in order to create a working product. This is one of the many reasons for why more and more companies are relying on ruby to create new, working prototypes and products.
