Arrays v Hashes

They're kind of a big deal..

2/18/2015

There are a couple of fundamental classes of objects in Ruby that make programming with the language a breeze. These are the Array and Hash classes.

Arrays are basically ordered lists of elements, complete with numerical reference values known as indexes. Arrays are useful in that they make it easy to store, retrieve, and manipulate elements according to their precedence or relatave importance. Typically, arrays consist of a homogenous set of a single class of object. You would have no real reason to want to store a heterogenous group of elements in an array if you can just as easily create multiple homogenous arrays of different object classes.


Hashes, however, differ from Arrays in that its not uncommon for them to consist of heterogenous set of objects. Think of classes as a named array, only instead of referencing specific elements by index value or precedence, you're referencing them by key-value pairs of elements. I would create a hash that has a set of keys that themselves reference specific values associated with their cooresponding keys. These values may be of the same type or may be entirely different from each other - it wouldn't matter. The only relation that these values have that is of any importance is the relationship they have with their reference point: their key. If I wanted to, I could even reference an entire array within a hash as a desired value when called by a string key.


There's a lot more flexibility with hashes than there are with arrays, and for this reason it's not uncommon for hashes to be used when dealing with security issues in rails.
