Regular Expressions

3/28/2015

There's a pattern that computer programs sometimes use to parse strings that they're working with. This pattern can be thought of as a 'regular' set of expressions used to check and manipulate information. In fact, the entire PERL scripting language is based on this set of rules, or regular expressions. Both ruby and Javascript use regex to perform a number of operations. Here I'm going to talk a little bit about how they work.

Regular expressions are built between a set of backslash delimiters / /. Inside the delimiters, any set of characters that you place will be used to build a regular expression "match."

For exmaple: Here's a simple ruby example asking to match a character 's' within a string variable holding the value of 'Here's my string':

string = "Here's my string"
=> "Here's my string"
string.match(/[s]/)
=> #<MatchData "s">


if string.match(/[s]/)
  puts "Found an s!"
end
Found an s!
=> nil

Since my regex found a match within the string variable for the character 's', I was able to execute my conditional logic asking the ruby program to output that I've found a string. You might notice that my string variable actually contains more than one 's'. With Regex, unless specified otherwise, the expression will match for the first successful match when parsing from left to right (in the same direction language is read and most programs parse information). If I wanted to match for more than one s and hold that information I could create another program that matches agains "groups" of a single 's' character:


if string.match(/(s)/).to_a.length < 2
   puts "There's only one s character!"
else
   puts "Found at least two s characters!"
end

Found at least two s characters!
=> nil

Using the parenthesis inside of the Regex delimiters will instruct that a program parse and then hold the matched data as independent variables. By converting my matched data into an array I was able to evaluate the number of s characters that were matched. If I had used square brackets as previously, I would only be able to check for a single s character at a time.

Regular expressions can get pretty tricky and difficult to understand if they become too cumbersome.I think its best to keep use of regular expressions limited unless absolutely necessary, as they can make code difficult to read and work with.
